# 종합점수 `combine_insight` (v2)

← [[00-스크리닝-점수-MOC|목차]]

## 왜 바꿨나

구버전은 시세 결측을 **50점 중립**으로 채우고, 인프라 미측정을 **0**, 마감 ≥30일을 **전부 10**으로 두었다.  
감정 할인이 40%만 넘어도 할인 점수가 100으로 포화되어, 전형 공매 목록이 **전부 44점**으로 뭉쳤다.

## 원칙

1. **결측은 제외** 후 가중치 재정규화 (가짜 중립 50 금지)
2. 할인 곡선이 20~70% 구간에서 **포화하지 않음**
3. 유찰 횟수 반영
4. 마감 D-40 vs D-90도 구분
5. 외부 API(시세·POI) 없이도 ingest 직후 점수 가능

## 성분 점수

| 성분 | 함수 | 비고 |
|------|------|------|
| 감정 할인 | `discount_to_score` | `18 + 100×ratio` (5~98) |
| 시세 할인 | 동일 | 결측 시 제외 |
| 인프라 | `infrastructure_score` | `None`=미측정 제외, `0`=측정·POI없음 |
| 긴급도 | `urgency_score` | half-life 18일, 하한 5 |
| 유찰 | `fail_count_score` | 0→22 … 8+→~88 |

## 기본 가중치 (있는 항목만)

| 항목 | 가중치 |
|------|--------|
| 감정 할인 | 0.32 |
| 시세 할인 | 0.28 |
| 인프라 | 0.18 |
| 긴급도 | 0.12 |
| 유찰 | 0.10 |

$$
\text{total} = \frac{\sum w_i s_i}{\sum w_i}\quad (s_i \text{ 존재하는 항목만})
$$
